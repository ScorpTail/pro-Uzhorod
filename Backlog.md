### 1 Перший спринт

1. Налаштувати backend-проєкт (Laravel, БД, міграції, базова структура).

2. Налаштувати frontend-проєкт (Nuxt, базова навігація).

3. Реалізувати API для новин (список + детальна новина).

4. Реалізувати сторінку новин з категоріями на фронтенді.

5. Реалізувати CRUD новин в адмін-панелі.

### 2 Другий спринт

1. Реалізувати головну сторінку з блоком останніх новин.

2. Реалізувати детальну сторінку новини.

3. Реалізувати сторінку довідника міських сервісів.

4. Підключити API для отримання списку новин та сервісів.

5. Реалізувати базові компоненти для повторного використання (карточки новини, карточки сервісів).

6. Реалізувати пагінацію або підвантаження новин на фронтенді.

### 3 Третій спринт

1. Реалізувати реєстрацію користувача (форма, валідація, підключення до API).

2. Реалізувати вхід користувача (login) через Laravel Sanctum.

3. Створити особистий кабінет користувача з базовою інформацією (профіль, редагування).

4. Зберігання токенів/сесій користувача на фронтенді (Pinia або composable useAuth).

5. Реалізувати можливість зміни пароля на фронтенді.

### 4 Четвертий спринт

1. Реалізувати сторінку створення та редагування міських сервісів в адмін-панелі.

2. Підключити API для створення, редагування, видалення та перегляду сервісів.

3. Додати можливість пошуку та фільтрації сервісів на фронтенді.

4. Реалізувати базову валідацію форм.

### 5 Пятий спринт

1. Реалізувати форму онлайн-звернення до міської ради.

2. Підключити форму до API (створення нового звернення).

3. Реалізувати вивід звернень у особистому кабінеті користувача (статус, дата).

4. Підключити список звернень для адмін-панелі через API.

5. Додати фільтри та сортування звернень на фронтенді.
