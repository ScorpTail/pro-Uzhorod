# Project Plan  
## Проєкт: pro-Uzhorod — веб-портал міста Ужгород

---

## 1. Опис проєкту

**pro-Uzhorod** — веб-портал для мешканців і гостей міста Ужгород, що надає доступ до новин, подій, міських сервісів та онлайн-звернень до міської ради.  
Проєкт орієнтований на зручність користування, масштабованість і стабільну роботу.

---

## 2. Цілі проєкту

- Забезпечити мешканців актуальною інформацією про місто.
- Надати можливість онлайн-звернень до міської ради.
- Створити зручну систему керування контентом для адміністраторів.
- Реалізувати проєкт із можливістю подальшого розширення.

---

## 3. Обсяг проєкту (Scope)

### Входить у проєкт:
- Довідник міських сервісів
- Реєстрація та вхід користувачів
- Онлайн-звернення до міської ради
- Адміністративна панель
- REST API (Laravel ↔ Nuxt)

### Не входить у першу версію:
- Мобільні застосунки
- Інтеграції зі сторонніми державними сервісами
- Повноцінна система сповіщень

---

## 4. Зацікавлені сторони (Stakeholders)

- Замовник / представники міської ради
- Кінцеві користувачі (мешканці, туристи)
- Команда розробки
- Адміністратори порталу

---

## 5. Технологічний стек

- **Frontend:** HTML5, CSS3, JavaScript, Vue.js, Nuxt.js
- **Backend:** PHP 8+, Laravel 10+
- **База даних:** MySQL
- **Аутентифікація:** Laravel Sanctum
- **Контроль версій:** GitHub

---

## 6. План реалізації (Етапи)

### Етап 1. Підготовка
- Аналіз вимог
- Проєктування архітектури
- Налаштування репозиторію

### Етап 2. Базова функціональність
- Реалізація новин
- Адміністративна панель
- REST API

### Етап 3. Користувацькі функції
- Реєстрація та авторизація
- Онлайн-звернення
- Особистий кабінет

### Етап 4. Оптимізація та завершення
- Безпека
- Продуктивність

---

## 7. План спринтів

### Sprint 1 (MVP)
- Налаштування Laravel і Nuxt
- API для новин
- Сторінка новин
- CRUD новин (адмін)

### Sprint 2
- Довідник міських сервісів
- CRUD сервісів
- Реєстрація та авторизація

### Sprint 3
- Онлайн-звернення
- Адмін-обробка звернень
- Покращення UI/UX

---

## 8. Ризики та управління ними

Основні ризики:
- Втрата даних
- Архітектурні помилки
- Вразливості безпеки
- Затримка фідбеку
- Розростання вимог

Управління ризиками:
- Регулярні бекапи
- Архітектурне рев’ю
- Sanctum, валідація, ролі
- Чіткий беклог і спринти

(Детально описано в Risk Matrix)

---

## 9. Якість та контроль

- Дотримання MVC та компонентної архітектури
- Code review
- Мінімальні тести для критичних функцій
- Перевірка продуктивності та безпеки

---

## 10. Критерії успіху проєкту

- Стабільна робота порталу
- Швидке завантаження сторінок
- Зрозумілий інтерфейс
- Можливість подальшого розширення
- Готовність до локального розгортання

---

## 11. Очікуваний результат

Функціональний, масштабований веб-портал міста Ужгород, що забезпечує доступ до інформації, міських сервісів та онлайн-взаємодію з міською адміністрацією.
